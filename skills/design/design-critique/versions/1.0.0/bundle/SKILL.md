# Design critique

Critique a screen or flow against these lenses; for each, state concern, severity, and a concrete fix.

1. Hierarchy — does the eye land on the primary action first? Is visual weight proportional to importance?
2. Consistency — components, spacing, and language match the system and the rest of the product.
3. Clarity — labels and copy are unambiguous; nothing relies on color alone.
4. Accessibility — contrast, focus, target size, motion-reduction, and screen-reader order.
5. States — empty, loading, partial, error, and success are all designed.
6. Content — concise, consistent voice, no lorem ipsum in a review build.

Separate blocking issues from polish, and never critique without a suggested direction.
