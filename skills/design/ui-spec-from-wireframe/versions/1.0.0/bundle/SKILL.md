# UI spec from a wireframe

Produce a spec a developer can build without guessing:

- Layout: regions, responsive behavior at each breakpoint, and overflow rules.
- Components: name each element, its variant/props, and the design-system component it maps to.
- Content: real copy or explicit placeholders; empty, loading, and error states for every data region.
- Interactions: what each control does, transitions, and optimistic vs blocking feedback.
- Data: fields shown, formats, truncation, and pagination.
- Accessibility: heading order, landmarks, focus order, and ARIA where semantics are insufficient.

Finish with an explicit list of open questions for the designer.
